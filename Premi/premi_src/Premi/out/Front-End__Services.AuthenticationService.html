logIn(email, password) → {Promise}

Metodo che richiede al back-end l'autenticazione dell'utente, utilizzando i valori ricevuti come parametro. Il metodo ritorna un oggetto Promise che può venire risolto o rifiutato. signUp(email, password, passwordCheck) → {Promise}

Metodo che, medinate $http, richiede al back-end la creazione di un nuovo account con le informazioni ricevute come parametro. Prima di effettuare la richiesta il metodo controlla che le due password ricevuto coincidano. Il metodo ritorna un oggetto Promise che può venire risolto o rifiutato. Nel caso la promessa venga rifiutata, verrà fornita come ragione un oggetto ErrorInfo contenente tutte le informazioni relative all’errore che si è verificato.
Parameters:
Name Type Description
email String Parametro che rappresenta la mail dell'utente che si vuole registrare.
password String Parametro che rappresenta la password dell'utente che si vuole registrare.
passwordCheck String Parametro che rappresenta la conferma della password inserita dall'utente.
